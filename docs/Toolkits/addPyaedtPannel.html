    PyAEDT面板安装工具 (addPyaedtPanels)

    工具概述

    addPyaedtPanels 是一个标准化的PyAEDT面板安装工具，提供图形用户界面(GUI)来简化在Windows环境下的PyAEDT配置和集成流程。该工具支持从多种来源安装PyAEDT，并自动将其集成到AEDT设计环境中。

    系统需求

    • 操作系统: Windows
    • Python版本: 3.10 或更高版本
    • AEDT版本: 2023.2 或更高版本
    • 网络环境: 支持在线和离线(wheel文件)安装

    工具位置

    PyAEDT面板安装工具位于Toolbox安装目录下：

    Toolbox安装目录/
├── UserLib/
│   └── Toolkits/
│       └── addPyaedtPanels/
│           ├── install_pyaedt_gui.py       （主程序 - GUI应用）
│           ├── refactor_utils.py           （共享工具函数）
│           └── 使用说明-郭永生-*.pdf      （使用文档）
└── ...

    主要功能模块

    1. 虚拟环境管理 (create_venv)

    • 功能: 创建专用的Python 3.10虚拟环境
    • 存储位置: %APPDATA%\.pyaedt_env\3_10\ (Windows)
    • 特点:
    • 自动检测既有环境，避免重复创建
    • 隔离的依赖管理，不影响系统Python
    • 支持AEDT内置Python直接使用

    2. PyAEDT安装方式

    方式一：在线安装 (install_pyaedt_online)

    • 优点: 获取最新版本，自动解决依赖
    • 缺点: 需要网络连接
    • 命令: 通过pip从官方源安装

    方式二：离线安装 (install_pyaedt_from_wheel)

    • 优点: 无需网络，安装速度快
    • 缺点: 需要提前下载wheel文件
    • 文件格式: .whl 或 .zip (自动解压)
    • 使用场景: 企业内网环境，多机部署

    3. AEDT面板集成 (add_pyaedt_to_aedt)

    • 功能: 将PyAEDT注册为AEDT设计面板
    • 效果: 安装后可在AEDT中直接使用PyAEDT API
    • 自动化: 调用PyAEDT官方安装脚本完成集成

    工作流程

    整体执行流程

    启动 install_pyaedt_gui.py
    ↓
检测AEDT环境
    ├─ 若在AEDT内执行
    │  └─ 使用AEDT自带Python (v261+ CPython 3.10)重新启动
    └─ 若独立执行
       └─ 使用系统Python 3.10+继续
    ↓
打开图形界面
    ├─ 方式选择 (在线/离线)
    ├─ 虚拟环境管理
    ├─ 安装进度监控
    └─ 结果反馈
    ↓
虚拟环境创建
    └─ 于 %APPDATA%\.pyaedt_env\3_10\ 创建venv
    ↓
PyAEDT安装
    ├─ 在线: pip install pyaedt[all]
    └─ 离线: pip install from wheel --find-links=<path>
    ↓
AEDT面板集成
    └─ 调用 ansys.aedt.core.extensions.installer.pyaedt_installer
    ↓
安装完成
    └─ 提示用户重启AEDT以应用更改

    AEDT根目录检测

    工具自动扫描环境变量查找AEDT安装路径：

    # 支持的环境变量格式 (大小写不敏感)
AnsysEM_ROOT241    # AEDT 2024.1
ANSYSEM_ROOT232    # AEDT 2023.2
ansysEM_root261    # AEDT 2026.1 (也支持)

    搜索策略

    1. 检查所有形式的 {AnsysEM_ROOT|ANSYSEM_ROOT|ansysem_root}+{3位版本号} 环境变量
    2. 验证对应路径的有效性
    3. 按版本号降序排列(最新版本优先)

    示例: - 若环境中同时存在 ANSYSEM_ROOT232 和 ANSYSEM_ROOT241，将选择 241 作为主版本

    GUI应用操作流程

    启动方式

    方式1: 直接在Toolbox菜单中调用 - 右键Toolbox → 选择"PyAEDT Panel Installer" - 自动检测AEDT环境并启动GUI

    方式2: 命令行启动

    python install_pyaedt_gui.py

    方式3: 在AEDT IronPython控制台执行

    exec(open(r"C:\...\addPyaedtPanels\install_pyaedt_gui.py").read())

    界面操作步骤

    1. 环境检测 - 程序自动检测AEDT版本和Python解释器
    2. 安装方式选择 - 选择"在线安装"或"离线安装"
    3. 虚拟环境 - 自动创建或使用既有环境
    4. 安装配置 - 设置安装选项(如依赖包)
    5. 进度监控 - 实时显示安装进度
    6. 完成反馈 - 成功/失败提示和日志

    多线程处理

    • UI线程: 保持界面响应性
    • 工作线程: 后台执行安装任务
    • 队列通信: 使用Queue进行线程间消息传递
    • 超时控制: 长时间操作自动超时预警

    故障排查

    常见问题

    问题 原因 解决方案
    "AEDT Python not found" AEDT版本过旧 升级AEDT到2023.2+
    "权限拒绝" 虚拟环境路径无写权限 修改%APPDATA%权限或更换安装路径
    安装超时 网络不稳定或源服务器缓慢 使用离线wheel安装或重试
    PersonalLib路径不存在 AEDT尚未创建默认目录 程序自动创建，无需手动处理

    日志查看

    • GUI日志: 安装界面中的"日志"标签页
    • 系统日志: %TEMP%\configure_pyaedt.py 执行结果
    • 错误追踪: 检查venv中的pip安装日志

    与Toolbox集成

    Python虚拟环境自动识别

    安装完成后，虚拟环境自动被Toolbox的Python检测机制识别：

    Toolbox启动时执行InitPythonEnv()
    ↓
检测 %APPDATA%\.pyaedt_env\3_10\Scripts\python.exe
    ↓
若存在 → 作为优先使用的Python
    ↓
后续所有Python脚本使用此环境执行

    注意事项

    ⚠️ 重要提示

    1. 不修改生成文件: install_pyaedt_gui.py 是从标准模板生成，不应手动修改
    2. AEDT重启: 安装PyAEDT panel后需要重启AEDT才能加载新面板
    3. 权限要求: 需要对%APPDATA%目录的写入权限
    4. 版本兼容性: 虚拟环境路径与AEDT版本无关，可跨版本重用
    5. 卸载说明: 删除 %APPDATA%\.pyaedt_env\ 目录即可完全卸载

    技术参考

    支持的wheel格式

    • 单个.whl文件
    • .zip压缩包(自动解压)
    • 目录结构: wheel文件和依赖包同目录

    venv内部结构

    .pyaedt_env\3_10\
├── Scripts\              (Windows执行文件)
│   ├── python.exe
│   ├── pip.exe
│   └── ...
├── Lib\site-packages\    (第三方包)
│   └── ansys\
│       └── aedt\
│           └── core\
└── pyvenv.cfg

    Python版本映射

    标识符 Python版本 AEDT版本
    3_10 Python 3.10.x 2024.1 - 2026.1
    3_9 Python 3.9.x 2023.2 - 2023.2R2
